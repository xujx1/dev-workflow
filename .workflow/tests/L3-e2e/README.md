## 目标

用固定 example 需求验证完整流程行为：

- **golden path**：完整跑通一个 example 需求，验证所有产物存在且格式正确
- **failure path**：缺文件/权限失败/覆盖率不足时流程正确停下
- **resume path**：模拟上下文中断后能否正确恢复到中断点继续执行

## 执行方式

L3 测试需要真实 LLM 调用，成本较高，建议只在版本发布前运行：

```bash

./run-e2e.sh
```
