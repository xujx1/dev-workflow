# L3 E2E Example 需求

本目录存放用于端到端测试的固定 example 需求。

## 要求

每个 example 应包含：
- `mrd.md` - 原始需求文档
- `expected/` - 预期产物目录结构
- `input.json` - 测试输入参数

## 示例结构

```
order-feature-example/
├── mrd.md                    # 原始 MRD
├── expected/                 # 预期产物
│   ├── prd.md
│   ├── tech-design.md
│   ├── test_spec.md
│   └── implementation/
└── input.json                # 测试输入
```
