## 一、背景与目标

### 1.1 背景

当前快递员签收流程缺少开箱验货环节，导致：
- 物品破损纠纷无凭证
- 签收后申诉率偏高（约 3.2%）

### 1.2 目标

- **主目标**：支持快递员当面开箱拍照签收，生成可追溯的签收凭证
- **次目标**：签收数据实时同步至履约域，驱动后续状态流转

### 1.3 成功指标

| 指标 | 目标值 |
|---|---|
| 签收凭证上传成功率 | ≥ 99.5% |
| 签收状态同步延迟 | P99 ≤ 3s |
| 开箱申诉率降低 | 目标 -30% |

## 三、功能变更

### 3.1 新增功能

**F1：拍照签收上传接口**

- 快递员在骑手 App 点击"拍照签收"，选择/拍摄照片
- 调用 OSS 预签名 URL 直传接口，获取上传凭证
- 照片上传成功后，提交签收请求（orderId + photoUrls + signTime）

**F2：签收记录查询接口**

- 支持按 orderId 查询签收记录（照片 URL 列表 + 签收时间 + 操作人）
- 支持按时间范围分页查询（后台管理）

**F3：签收状态同步**

- 签收完成后，通过 MQ 发布 `ORDER_SIGNED` 事件，通知下游系统

### 3.2 修改功能

- `GET /delivery/order/{orderId}/status` 接口新增 `signPhotoUrls` 字段

## 四、验收标准（AC）

| ID | 场景 | 预期结果 |
|---|---|---|
| AC-01 | 快递员提交签收（含3张照片） | 签收记录入库，MQ 发布成功，返回 signId |
| AC-02 | 重复提交同一 orderId + signTime | 返回已有签收记录，不重复写入 |
| AC-03 | 照片数量超出限制（>5张） | 返回 400，提示"照片数量超过限制" |
| AC-04 | OSS 上传失败后提交签收 | 签收接口拒绝，要求重新上传照片 |
| AC-05 | 查询签收记录 | 返回照片 URL 列表，URL 带签名有效期 1h |
