# 安全规范

## 提交前强制检查

在任何 `git commit` 之前：

- [ ] 无硬编码密钥（API Key、密码、AK/SK、Token）
- [ ] 无硬编码 IP 地址或内网域名
- [ ] 所有用户输入已做有效性校验
- [ ] SQL 使用参数绑定，无字符串拼接（禁止 `${}` — B2）
- [ ] 无 `SELECT *`（B1）
- [ ] 敏感信息未写入日志（密码、手机号、身份证号）
- [ ] RPC/HTTP 超时已配置

---

## 密钥管理规范

- **禁止**在源代码中硬编码任何密钥、密码、证书
- **必须**使用环境变量或配置中心（如 Apollo/Nacos）
- **启动时**验证必填密钥是否存在，缺失则拒绝启动
- **泄露处理**：立即轮换已泄露密钥，全量扫描类似硬编码

---

## 安全响应流程

发现安全问题时：

1. **立即停止**当前工作
2. 使用 **security-reviewer** 子代理做全量扫描
3. 修复所有 CRITICAL 问题后再继续
4. 若有密钥泄露，执行轮换流程
5. 审查整个代码库的类似模式

---

## Java 特有安全检查

| 场景 | 检查项 |
|------|--------|
| MyBatis SQL | 禁止 `${}` 拼接（SQL 注入），只用 `#{}` |
| 用户入参 | Controller 层必须 `@Valid` 或手动校验 |
| 文件上传 | 校验文件类型、大小、路径（防路径遍历） |
| 序列化 | 避免直接反序列化不可信数据 |
| 日志脱敏 | 手机号、身份证号、银行卡号打 `*` 脱敏后记录 |
