# 流程耗时统计规范

> 本文档汇总所有斜杠命令（`/01` ~ `/08`）的计时规范，供 Agent 执行时统一参考。

## 通用规则

1. 每个命令执行时，**必须**在步骤/Phase 的开始和结束处记录时间戳（格式 `HH:MM:SS`），并在全部步骤完成后输出耗时报表
2. 若某步骤被跳过，耗时列填 `—（跳过）`
3. 报表**必须**持久化到 `{feature_dir}/execution-state.md` 的 `## 时延统计` 章节下，子章节名为 `### /XX-命令名 耗时报表`
   - 子章节已存在：**覆盖**该子章节全部内容，不影响同章节下其他子章节
   - 子章节不存在：在 `## 时延统计` 末尾**追加**；若 `## 时延统计` 章节本身不存在，先创建章节再追加
   - 若 `execution-state.md` 文件不存在，跳过写入，仅在会话中输出报表

### 实时追加规则（每个环节 / Agent 独立写入）

> 上述规则 3 描述的是命令**全部完成**后的最终报表写入。本节补充更细粒度的实时追加要求。

**每个环节（步骤 / Phase）和每个子 Agent 完成后，必须立即将自己的耗时记录追加到 `execution-state.md`，不得等到命令全部结束才一次性写入。**

具体要求：

1. **环节粒度**：每个步骤（S1/S2/…）或 Phase（P1/P2/…）完成时，立即将该行的耗时数据写入 `execution-state.md` 对应子章节的表格中
2. **Agent 粒度**：每个子 Agent（如 `tdd-test-spec-agent`、`java-impl-agent` 等）完成时，立即将该 Agent 的内部步骤明细子表追加到 `execution-state.md` 对应子章节中
3. **写入方式**：采用追加模式——在对应子章节的表格末尾逐行追加，而非每次重写整个子章节。若子章节或表格尚不存在，先创建骨架再追加首行
4. **最终汇总行**：`**合计**` 行仅在命令全部步骤完成后写入（因为合计依赖所有步骤完成）
5. **回流场景**：每次回流的环节完成时同样立即追加，行尾标注 `[回流第N次]`
6. **容错**：若 `execution-state.md` 文件不存在或写入失败，不阻塞主流程，仅在会话中输出该环节耗时

**单环节追加模板：**

```
# 环节完成时，向 execution-state.md 的 ### /XX-命令名 耗时报表 子章节追加一行：

| {步骤编号}: {步骤名称} | {开始时间} | {结束时间} | {耗时} | {备注} |

# 若该环节对应一个子 Agent，同时追加 Agent 内部步骤明细子表：

#### {步骤编号} {agent-name}

| 步骤 | 开始时间 | 结束时间 | 耗时 | 备注 |
|------|---------|---------|------|------|
| S1: xxx | HH:MM:SS | HH:MM:SS | Xm Xs | |
| S2: xxx | HH:MM:SS | HH:MM:SS | Xm Xs | |
| ...

# 全部环节完成后，追加合计行：

| **合计** | — | — | **Xm Xs** | |
```
