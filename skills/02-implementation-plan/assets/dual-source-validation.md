# 02-implementation-plan — 知识库验证模式

> 由 `SKILL.md` 按需 Read。

---

## 架构

```
MRD ──→ app-knowledge-base (事实基准) ──→ PRD → 技术方案
             ↓ 接口清单/核心逻辑
          以应用库为唯一事实源
```

---

## 完整性检测（Step 0）
    - 检测 `CONTEXT.md` 是否存在
    - 检测 `KB_FRESHNESS.md` 是否存在
    - 若任一缺失，提示用户选择：
        - A. 补齐缺失文件（推荐）
        - B. 跳过，使用现有知识库

---

## 冲突解决规则

- 应用知识库为唯一事实源，技术方案与之冲突时以应用库为准
