# PRD 模板（一~七章）

> Stage 1 PRD 生成必须使用本模板。PRD 面向产品/业务评审，不写实现类、包名、方法名或代码片段。

## 一、背景

说明问题背景、业务现状、影响范围和量化损失。

## 二、目标

说明本次需求要达成的业务目标、边界范围和明确不做事项。

## 三、角色 / 场景

用业务角色和场景描述需求，不使用技术实现口径。

## 四、功能变更

列出功能编号、功能名称、功能描述、优先级。

## 五、业务规则

描述判断规则、处理规则、排除范围和业务流程。流程图必须使用 Mermaid `flowchart`。

## 六、验收标准

使用可验证的 AC 条目描述验收标准。

## 七、边界 / 待确认

列出边界、依赖协作事项和仍需确认的问题。不得使用 `❓` 标注。

## 生成元数据尾注

PRD 末尾必须追加：

```markdown
> **生成元数据**
> 工具：dev-workflow vX.Y.Z | Skill: prd-gen vX.Y.Z
> 生成时间：YYYY-MM-DD
> 知识库快照：YYYY-MM-DD
```
