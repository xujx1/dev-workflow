# 技术方案模板（紧凑版）

> Stage 2 技术方案生成至少满足本模板。若业务系统要求完整技术模板，可使用 `agents/tech-design/assets/tech-design-template.md`，但确认门仍会检查核心主题。

## 一、需求概述

说明需求背景、目标、澄清结论和范围。

## 二、接口变更

说明是否涉及 HTTP / DUBBO / MQ / 外部契约变更。不涉及也必须明确写明。

## 三、核心修改

说明核心逻辑修改点、数据读取路径、幂等和兼容性。

## 四、时序图

必须使用 Mermaid，不允许 ASCII art。

## 五、影响分析

说明代码影响、业务影响、上下游影响、风险等级。

## 六、工时预估

给出任务拆分与人日估算。

## 七、测试要点

覆盖单元测试、集成测试、回归测试和异常场景。

## 八、风险评估

列出风险项、等级和应对措施。

## 九、备注

记录评审注意事项和协作依赖。

## 附录I

需求拆解。

## 附录II

变更影响分析，优先使用 GitNexus；不可用时记录 fallback。

## 附录III

场景扩展分析。

## 附录IV

多视角架构分析。
