# MRD 澄清详情

## 不确定项类型

| 类型 | 示例 | 识别方式 |
|------|------|----------|
| 接口歧义 | MRD 提及"订单查询接口"，系统中存在多个订单查询入口 | 知识库中接口清单对照 |
| 字段歧义 | MRD 提及"用户类型"，系统中有多个用户分类维度 | 知识库中数据结构对照 |
| 逻辑歧义 | MRD 描述"审核通过后发货"，代码支持预发货 | 知识库中核心流程对照 |
| 边界不清 | MRD 未明确异常分支处理方式 | MRD 文本分析 |
| 涉改范围 | MRD 涉及多个模块，优先级不明确 | 知识库中模块依赖分析 |

## 问答模板

```markdown
## MRD 澄清问题

发现以下需要确认的问题，请逐一回答（已从知识库预填的答案，直接确认即可）：

### 1. {问题类别}
**问题**：{具体问题描述}
**知识库参考**（如有）：{知识库中找到的相关信息}
**预填答案**（如有）：{基于知识库的建议答案，无则空}
**选项**（如适用）：
- A. {选项A}
- B. {选项B}
**你的确认/补充**：_

### 2. {问题类别}
**问题**：{具体问题描述}
**你的回答**：_

---

请回复以上问题的答案，格式如：
1. 确认（或填写具体答案/修改内容）
2. {具体答案}
```

## 澄清版 MRD 写入格式

> 使用 Bash heredoc 写入，避免 Write 工具被 Hook 阻止（见 `rules/common/large-doc-writing.md`）。

```bash
mkdir -p "{feature_dir}"
cat > "{feature_dir}/mrd-clarified.md" << 'EOF'
# {MRD 原标题}

{MRD 原文内容}

---

## 澄清记录

> 澄清时间：{timestamp}

| # | 问题 | 答案 |
|---|------|------|
| 1 | {问题1} | {答案1} |
| 2 | {问题2} | {答案2} |
EOF
```

## 多写分发（仅多域模式）

仅当 `apps[]` 或 `all_apps[]` 不为空时执行。

**6-A：mrd-original.md 全量多写**（`all_apps[]` 不为空时）

遍历 `all_apps[]`，对每个 app 的 `feature_abs_path` 执行：
- 若 `feature_abs_path == feature_dir` → 跳过（已有）
- 否则：`mkdir -p "{feature_abs_path}" && cp "{feature_dir}/mrd-original.md" "{feature_abs_path}/mrd-original.md"`

**6-B：mrd-clarified.md 域内多写**（`apps[]` 不为空时）

遍历 `apps[]`，对每个 app 的 `feature_abs_path` 执行：
- 若 `feature_abs_path == feature_dir` → 跳过（已有）
- 否则：`mkdir -p "{feature_abs_path}" && cp "{feature_dir}/mrd-clarified.md" "{feature_abs_path}/mrd-clarified.md"`

**落盘验证**：检查所有目标路径的文件均存在，任一缺失则重试复制（最多 2 次）。

## 约束

- **一次性提问**：所有问题必须一次性提出，不得逐个追问
- **知识库预填而非静默跳过**：知识库能解答的疑点，以预填答案形式展示并要求用户确认；不得因知识库已有答案而跳过向用户展示
- **零疑点时明确告知**：若确实未识别到任何疑点，必须明确输出「无需澄清」结论，不得静默跳过
- **禁止修改原文**：澄清内容追加到 MRD 末尾，不删除原文任何部分
- **阻塞执行**：未收到用户回答前，不得继续执行后续 Agent
