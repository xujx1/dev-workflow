# 需求拆解指南

> 技术方案阶段读取本文件，基于 PRD 执行需求拆解，追加到 `{feature_dir}/tech-design.md` 末尾的 `附录I：需求拆解`。

---

## 拆解原则

1. **Story = 可独立交付的最小用户价值单元**，一个 Story 不跨多个核心业务对象
2. **AC = 可测试的验收条件**，每条满足 Given-When-Then 结构
3. **依赖关系显式声明**，环形依赖不合法
4. **负责方明确**：后端 / 前端 / 全栈 / 配置

---

## 拆解质量检查

| 检查项 | 通过标准 |
|--------|---------|
| Story 独立性 | 每个 Story 可独立上线，不强依赖同批 Story |
| AC 完整性 | 每个 Story 至少 2 条 AC，满足 Given-When-Then |
| 依赖无环 | 依赖关系图为有向无环图（DAG） |
| 优先级对齐 | Story 优先级与 PRD 功能优先级一致 |
| 任务可估算 | 每个开发任务可在 0.5~3d 内完成，否则继续拆 |
| 风险项列出 | `[需确认]` 标注全部收录到「待确认事项」 |
